## Troubleshooting errors when loading events

**Source:** [https://simple-history.com/support/load-events-error/](https://simple-history.com/support/load-events-error/)

Sometimes the log does not load correctly and it may be hard to understand why it’s not working.

You have much likely come here because you encountered a message like this while trying to load the log with events:

![](https://simple-history.com/wp/wp-content/uploads/2024/12/error-loading-events-details-1024x911.png)

There can be many different reasons for this error. It may be temporary or permanent. On this page we’ll try to give you some tips and help on how to fix this problem.

### Common solutions

- Reload the page
- Check the server logs for errors
- Check the web browser console for errors
- Check the database server logs for errors

### Common problems

Error 403: Cookie check failed (rest_cookie_invalid_nonce)
This can happen if you leave the history page open for a long time, without leaving it or reloading it. WordPress has a built in security check that causes the REST API calls to be invalid after a long period.

Solution: Reload/refresh the page using your browser reload-button and the problem should go away.
