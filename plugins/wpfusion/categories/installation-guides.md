## How to Connect Sender.net to WordPress

**Source:** [https://wpfusion.com/documentation/installation-guides/how-to-connect-sender-net-to-wordpress/](https://wpfusion.com/documentation/installation-guides/how-to-connect-sender-net-to-wordpress/)

### Overview

In this guide, you will learn how to install WP Fusion and set up a direct connection to integrate your WordPress website with your Sender.net account.

To get started install and activate the WP Fusion plugin, then navigate to Settings » WP Fusion.

From the ‘Select Your CRM’ list choose ‘Sender’.

Enter your Access Token. You can create an Access Token by navigating to Sender.net Settings » API Access Tokens. Then click ‘Connect’ to initialize the connection.

Once the connection is completed, click Save Changes below to save the changes.

That’s it! Your site is connected, and new WordPress users and customers will automatically be synced with Sender.net in real time.

- For exporting historical data from before you installed WP Fusion, check out the Exporting Data guide.
- For mapping and syncing additional custom fields with Sender.net, see the Syncing Contact Fields guide.
- For using Sender.net groups to track activity in WordPress and personalize the user experience, check out the Working with Tags and Restricting Access to Content guides.

## How to Connect Omnisend to WordPress

**Source:** [https://wpfusion.com/documentation/installation-guides/how-to-connect-omnisend-to-wordpress/](https://wpfusion.com/documentation/installation-guides/how-to-connect-omnisend-to-wordpress/)

### Overview

In this guide, you will learn how to install WP Fusion and set up a direct connection to integrate your WordPress website with your Omnisend account.

From the ‘Select Your CRM’ list choose ‘Omnisend’.

- For mapping and syncing additional custom fields with Omnisend , see the Syncing Contact Fields guide.
- For using Omnisend tags to track activity in WordPress and personalize the user experience, check out the Working with Tags and Restricting Access to Content guides.

### Connect your store to Omnisend

To use automations, first you need to enable Site tracking in WP Fusion.

Go to your Omnisend account, navigate to Automation, and click ‘Connect your store’.

Select ‘Other’, enter your store URL, and click ‘Connect Store’.

Then click ‘Verify and Connect’.

This will check if the tracking snippet is installed and then the store is connected.

**Note:** In some cases it can take up to 24 hours to validate the tracking script and enable automations in Omnisend. If Omnisend was unable to automatically verify your store, you will see a banner across the top of your account.

If your store does not appear validated after 24 hours, please [contact Omnisend support](https://www.omnisend.com/contact-us/support/).

#### Video

### Omnisend tags

At the moment it’s not possible to remove tags via the Omnisend API.

To get around this, when a tag is removed WP Fusion will update a custom property on the subscriber with the tag name, and the value set to false.

When loading tags, WP Fusion will also check the subscriber’s custom property values to generate an accurate list of the subscriber’s tags.

This makes Omnisend fully functional with all of WP Fusion’s features and integrations. If you would prefer not to see this custom properties in your account, you can also create an automation to remove the associated tag when the custom property is updated.

### Event Tracking

Custom events tracked by WP Fusion show up in each subscriber’s [activity feed](https://support.omnisend.com/en/articles/2760491-contact-profile).
Clicking Event Details shows the details for a specific event. These values can then be used in automations.
WP Fusion’s Event Tracking Addon can send events in real time from supported plugins to Omnisend. For more information, see the documentation.

Finally, click on ‘Connect’.

To finalize the process click on ‘Save Changes’.

### Custom fields

Custom properties in Customer.io can be managed directly in WP Fusion.

When configuring your field mappings via the Contact Fields settings, simply type in the desired property name, and click the prompt to (add new).

The data will be synced automatically to the custom property.

#### Tag format

There is an option when setting up WP Fusion with FluentCRM over the REST API to choose the Tag Format, either “IDs” or “Slugs”.

This is due to a quirk in the early versions of the FluentCRM API. When we initially developed the integration, it was not possible to use tag IDs over the API, and so WP Fusion used the tag slugs (a slug is an internal text value for a tag, like my-tag-name).

For all new installs of WP Fusion, we recommend using Tag IDs for the best compatibility, as well as to allow easily migrating FluentCRM from the same site to a subdomain.

However, for existing installs, it’s recommended to keep the tags stored as slugs, otherwise you will need to set up your access rules, products, and other tag-based settings again from scratch.

There is no difference between IDs and slugs in terms of features or performance, but we are encouraging IDs for new sites to ensure consistency with our other CRM integrations.

**Note:** It is possible to switch the setting between IDs and Slugs after setting up WP Fusion. After doing this, click Refresh Available Tags and Fields to load the tags in the new format, and run a [Resync Tags for All Users operation](https://wpfusion.com/documentation/tutorials/batch-operations/#resync-tags) from the Advanced settings tab to load the updated tag format into each user’s WordPress record.

### Webhooks

Using webhooks in FluentCRM automations, you can automatically sync tag changes and contact updates from FluentCRM back to WordPress. You can even import new users based on FluentCRM contacts.

For more information, see the documentation on FluentCRM Webhooks.

### Event Tracking

Events tracked by WP Fusion are displayed on the subscriber’s activity feed in FluentCRM.
WP Fusion’s Event Tracking Addon can send events in real time from supported plugins to Encharge. For more information, see the documentation.

### Allow duplicate phone numbers

By default HighLevel does not allow two contacts to have the same phone number.

If you try to create a contact over the API, and another contact in HighLevel already has the same phone number, you will get an error This location does not allow duplicated contacts, and no data will be saved.

For this reason we recommend allowing duplicate contacts in HighLevel at Settings » Business Profile » Allow Duplicate Contact.

WP Fusion will always perform a lookup based on email address before creating a new contact, to ensure no duplicate contacts are created with the same email.

By allowing duplicate contacts in the settings, this will allow two contacts with different email addresses to have the same phone number (for example two members of an organization or household).

#### API Limits

Keap limits each Service Account Key to a maximum of 10 queries per second, 240 API calls per minute, and 30,000 per day.

Most sites won’t come up against these limits, but have sometimes seen customers experience API throttling on high volume ecommerce or membership sites.

WP Fusion has three strategies in place to avoid API throttling:

1. When exporting or resyncing data in bulk, WP Fusion will wait two seconds between each operation to allow the API to “cool down” and reset. This means you can export thousands of customers or orders without hitting the API limits, it will just take longer.
2. If WP Fusion receives an API response that indicates the API is being throttled, it will first wait two seconds, and then try the API call agaib.
3. If this fails, WP Fusion will look for a Backup Service Account Key. If found, WP Fusion will switch to the backup key for all API requests until 12am UTC, when the API quotas are reset.

#### Backup Service Account Key

To add a backup Service Account Key, first set up WP Fusion with your primary SAK, and initialize the connection.

Then, following the same steps you used to create the first SAK, create a second one, and paste it into the Backup Service Account Key field.

If your first SAK exceeds 30,000 requests in one day, WP Fusion will fall back to the backup SAK until the API quotas reset at 12am UTC.

You will see a notice at the top of the WP Fusion settings if the backup SAK is currently in use.

The switch to the backup SAK will also be recorded to the WP Fusion activity logs.

#### Developers

WP Fusion uses a transient to keep track of the backup SAK. If WP Fusion finds a value using get_transient( 'wpf_keap_backup_key' );, this will be used for the API key.

For more complex sites, you could create your own backup SAK logic by overriding this transient with your own custom logic.

For example you could create a pool of 12 SAK’s, and create a function using the transient_{$transient} filter on the get_transient() to automatically rotate through the 12 SAKs every two hours.

This and more complex implementations are possible with some creative thinking, but are beyond the scope of this article.

(The Use Count counts the number of times the access token has been refreshed, and doesn’t contribute to the connected sessions limit).

In this video, you will learn how to install WP Fusion and set up a direct connection to your Intercom account.

To get started install and activate the plugin, then navigate to settings -> WP Fusion.

From the ‘Select Your CRM’ dropdown choose ‘Intercom’.

To enable the connection we need to locate your Access Token.

This can be found by clicking this link and logging into your Intercom account.

Copy the access token and paste it into the appropriate field in WP Fusion.

Then click on ‘Test Connection’.

WP Fusion will now establish a direct connection to your Intercom account and import your tags and custom fields.

To finalize the process click on ‘Save Changes’.

## How to Connect Drip to WordPress

**Source:** [https://wpfusion.com/documentation/installation-guides/how-to-connect-drip-to-wordpress/](https://wpfusion.com/documentation/installation-guides/how-to-connect-drip-to-wordpress/)

In this video, you will learn how to install WP Fusion and set up a direct connection to your Drip account.

To get started install and activate the plugin, then navigate to settings -> WP Fusion.

From the ‘Select Your CRM’ dropdown choose ‘Drip’.

We will need to locate two pieces of information:

1. Your ‘Account ID’
2. Your ‘API token’

To find these credentials login to your Drip account and click on the ‘Settings’ menu in the sidebar.

Click on ‘Account’

And open the ‘General Info’ tab

Then scroll down and locate your Account ID.

Copy the ID and paste into WP Fusion.

To locate the API key, return to Drip, click on the ‘Settings’ menu in the sidebar.

Click on ‘User Settings’

Scroll down to the bottom and copy the API token and paste into the appropriate field in WP Fusion.

Finally, click on ‘Connect’.

WP Fusion will now establish a connection to Drip and import your tags and custom fields.

To finalize the process click on ‘Save Changes’.
